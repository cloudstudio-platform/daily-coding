当谈到Cloud Studio浏览器插件的优势时，**最显著的就是它的便捷性**。通过安装Cloud Studio浏览器插件，用户可以在浏览器中直接打开Cloud Studio的开发环境，无需切换到其他应用程序，从而提高了开发效率。

另一个优势是**插件对于 GitHub、CODING 等仓库的感应能力**。譬如，当用户打开GitHub仓库时，插件会自动适配当前地址以及代码里的环境定义，并提供按钮一键打开。这意味着用户可以直接从 GitHub 仓库中直接进入 Cloud Studio 的开发环境，无需手动输入地址或配置环境，从而节省了时间和精力。

![001.png](https://help-assets-1257242599.cos.ap-shanghai.myqcloud.com/enterprise/03-plug-in/001.png)

目前 Cloud Studio **支持 Chrome 和 Edge 浏览器两种插件**，同时我们也提供了离线安装包。

- 在线安装步骤：

[Cloud Studio 浏览器插件 | Cloud Studio](https://cloudstudio.net/docs/guide/browser-ext.html#%E5%9C%A8%E7%BA%BF%E5%AE%89%E8%A3%85%E6%AD%A5%E9%AA%A4)

- 离线安装步骤：

[Cloud Studio 浏览器插件 | Cloud Studio](https://cloudstudio.net/docs/guide/browser-ext.html#%E7%A6%BB%E7%BA%BF%E5%AE%89%E8%A3%85%E6%AD%A5%E9%AA%A4)


此外，Cloud Studio 浏览器插件还提供了**多种语言的代码高亮显示功能**，让用户的代码更加清晰易读。同时，插件**还支持多种开发语言和框架**，包括 Java、Python、Node.js等，可以满足不同用户的开发需求。

我们也非常欢迎您体验 Cloud Studio 徽标。您可以在代码仓库的 README.md 中放置Run on Cloud Studio 的代码，如下，替换「你的仓库地址」即可。


``` 
[![CloudStudioTemplate](https://cs-res.codehub.cn/common/assets/icon-badge.svg)](https://cloudstudio.net#你的仓库地址)
```

譬如，以 [Nocalhost 开源仓库为例：GitHub - nocalhost/nocalhost: Nocalhost is Cloud Native Dev Environment.](https://github.com/nocalhost/nocalhost)

![002.png](https://help-assets-1257242599.cos.ap-shanghai.myqcloud.com/enterprise/03-plug-in/002.png)

总之，Cloud Studio 浏览器插件的优势在于其便捷性、智能适配功能、多语言支持以及实用功能。如果您是一名开发人员，不妨尝试安装 Cloud Studio 浏览器插件，提高您的开发效率。
