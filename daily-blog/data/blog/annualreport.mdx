时光飞逝，一转眼 2022 年只剩下最后几天。回顾 2022 年，Cloud Studio 产品经历了很多的迭代，给大家带来了很多亮点，**在此辞旧迎新之际，与大家一起分享 Cloud Studio 的亮点时刻。**

### **2022 年产品亮点回顾**

**DeployKit 云部署套件**

今年初，我们结合腾讯云 Serverless 技术，把前端开发项目、静态建站项目开发的部署复杂度降到最低。在产品上我们坚持极简的部署产品交互，但背后它却可以智能适配数十款的前端建站项目。开发完，在 Cloud Studio 预置的页面中看到开发效果后，一键即可完成线上部署。

[_👉  Cloud Studio - DeployKit_](https://cloudstudio.net/deploykit)

 **全新预览调试功能**



前端开发的用户群体中，特别需要所见即所得。我们做了几个优化：
1\. 端口智能探针，自动识别新端口；
2\. 快速从端口处找到进程号、快速打开预览地址；
3\. 针对预览地址，我们做了手机适配效果。


![](https://help-assets.codehub.cn/enterprise/cloudStudio/007.png)

 **新增微信登陆**

手机掏出来扫一下，你就可以进行登陆。非常方便看他人分享的模板或者是邀请协作。


![](https://help-assets.codehub.cn/enterprise/cloudStudio/008.png)

 **全新自研 Metawork 协作套件**


Cloud Studio 率先创新的提出了在开发领域下的**协同能力**，让协作开发像协作文档一样丝滑流畅。通过协同邀请团队好友进入你的开发工作空间，来帮助或者协助一起结对编程、解决问题、代码接龙等诉求。


Cloud Studio 协作套件推出**代码协作**，支持多光标写代码，同时还支持光标跟随、滚轮跟随，非常适合上课模式和代码投屏；我们还支持终端多光标跟随，解决尺寸不同导致的乱码问题，我们还支持留言板，圈点代码讨论；我们和音视频团队一起，还支持语音、视频能力，为整个疫情的远程办公助力。


Cloud Studio 协作套件还**保证安全性**，为每一次邀请链接提供了安全保障和功能性提醒，譬如踢出该成员等。


**我们非常重视和开发者的反馈意见，并积极主动推动实现落地；我们陆续支持了：**

*   IPad 最新版本 + 妙控键盘的开发流畅性

通过对产品的前端优化，提升 iPad 触控操作 Cloud Studio 界面的体验，现已支持妙控板双指滑动等硬件匹配操作。

*   IDE 内的绿点的快捷操作

点击绿点，即可列出常用操作，避免返回到控制台进行其他操作的步骤。

![](https://help-assets.codehub.cn/enterprise/cloudStudio/009.png)

*   内存不够时的预警

当用户使用内存达到 80% 时报警，用户使用内存满，进程被强制 kill 后提示用户。

*   GIT 代理

来自于用户反馈：在个人设置下，可以为 Git 访问进行代理服务器的设置，以保证一些特殊场景下，可以正常获取代码。特别是在企业自建 VPN，在外网访问需要设置代理的一些场景下非常有用。

![](https://help-assets.codehub.cn/enterprise/cloudStudio/010.png)

*   永不休眠及定时休眠能力

新增 IDE 内，长时间不操作之后的提示并切断的能力，可设定 5/10/20 分钟，以及永不休眠。

![](https://help-assets.codehub.cn/enterprise/cloudStudio/011.png)

*   **匿名体验用户可以看到更多模板啦**

我们支持在未登陆下，也可以访问控制台，并可以体验控制台下的各种示例库了。方便用户直接学习访问，快速了解 Cloud Studio产品带来的代码乐趣。当然部分功能还是需要注册才能体验的哦。

![](https://help-assets.codehub.cn/enterprise/cloudStudio/012.jpeg)

*   HTML 文件预览能力

很多开发者是刚刚学习前端开发，也或者是做大数据报表，通过演算后会生成 HTML 报表。为了满足这类用户诉求，我们支持 HTML 的文件预览能力。内置了 HTML 预览服务器，让文件快速达到真实效果。

 **增加了多款主流框架和示例库**

越来越多的前端开发者都在使用符合 JamStack 类型的框架。我们在控制台内置了多款主流的建站框架、前端框架、后端框架及对应的示例库。可以非常方便一键打开运行效果。我们还支持了 Flutter 框架，让云端开发移动 App 成为一种可能。


![](https://help-assets.codehub.cn/enterprise/cloudStudio/013.png)

 **登录授权关联第三方仓库**

 **服务商能力**

支持导入第三方仓库，深度融合 CODING 仓库。我们发现很多用户其实存在多账号的数据，特别是个人仓库、团队仓库、企业仓库，于是我们在年中推出多账号登陆关联授权第三方仓库服务商能力。

 **自定义Cloud Studio**

 **开发环境代码化**

软件工程中，一切皆代码，通过 IaC（Infrastructure as Code)，我们可以通过代码来定义开发空间的启动、环境、插件等等步骤的信息，并组装到 workspace.yaml 这个文件中。

 [_👉 Cloud Studio 高阶玩家：强大的 YAML 模板_](https://mp.weixin.qq.com/s?__biz=MzkwODMzNTExMw==&mid=2247484041&idx=1&sn=23615ccace11b2da8323d286c9cce09f&scene=21#wechat_redirect)

我们在 11 月还支持了 Postgres 的接入，可以定义开发环境下的 PG 配置。

同时还支持环境镜像的版本展示、启动加载命令等优化。

**基础镜像热替换**

选错环境？但是开发到一半，怎么办？没事，Cloud Studio 创新的支持了基础镜像替换。你可以通过底部条进行基础镜像切换，我们会为你保留所有文件不丢失。同时热替换也会和当前的代码配置 workspace.yaml 进行联动。

 **产品更标准化的建设**

今年初我们发起了一场标准化产品道路建设的治理，在几个月的努力下初见成果。

*   产品离线安装更标准：我们推出了简单免费镜像安装，通过下面命令即可在本地一键拉取运行。


```

docker run -it -p 8080:8080 codingopen/cloudstudio-full:latest --auth=none

```

另一种方式，是基于 K8s helm chart 的安装，支持广大私有化企业。


*   支持 ARM 架构的云主机上。



 **代码的 Git 操作优化升级**


代码版本化是一个很重要的开发阶段，我们从下半年起开始针对性的优化这块的交互。我们陆续支持优化了如下几个亮点功能：

*   对 CODING 仓库深度结合，支持一键转存 CODING 仓库。如果是用 CODING 账号登陆 Cloud Studio，可以免登。

*   git-commit 插件功能优化，支持创建分支，undo commit 操作；支持 commit/ 分支跳转至 coding 仓库页面；支持 GIT 版本、GIT 提交、GIT 标签列表的对比记录查看。

*   支持新的 Rebase Editor，帮助你可视化的去更新合并提交。


![](https://help-assets.codehub.cn/enterprise/cloudStudio/014.png)
